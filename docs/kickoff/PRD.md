# PRD V2 — mike-product-calc（蜜可诗产品经营决策台）

> 升级版 Kickoff：2026-04-11
> 数据源：`蜜可诗产品库.xlsx`（Gelato 84SKU + 雪花冰 10SKU + 饮品 1SKU）
> 产品定位：**目标毛利驱动的产品与原料定价决策台**

---

## 1. 背景与目标

### 业务背景
蜜可诗当前用 Excel 管理产品成本与定价，缺乏系统化的：
- 单品 / 组合毛利分析
- 目标毛利率反推原料定价
- 灵活调价模拟
- 按日期的生产计划与备料
- 选品决策支持

### 产品定位
**本地优先的产品经营决策工作台**——核心不是算数，而是让运营者能快速做出"卖什么、备多少、原料什么价、赚多少、卡在哪里"的判断。

### 可量化目标
- 输入任意产品组合，< 3 秒输出完整利润 + 原料需求结果
- 输入目标门店毛利率，< 3 秒输出推荐原料定价方案
- 输入 7 天生产计划，即时输出每日备料计划 + 采购建议
- 所有结果可导出 CSV，支持与 Excel 交叉验证

## 2. 五大核心模块

### 模块 1 — SKU 毛利分析
**回答：这个产品赚不赚钱**

功能：
- 按 Gelato / 雪花冰 / 饮品分类浏览所有上线 SKU
- 显示：定价、门店成本、门店毛利率、工厂毛利率
- 成本拆解瀑布图（主原料、配料、包材）
- 双口径切换（出厂口径 / 门店口径）

数据来源：
- `产品毛利表_Gelato` / `产品毛利表_雪花冰`
- `产品成本计算表_Gelato`
- `产品出品表_Gelato`

---

### 模块 2 — 目标毛利率反推原料定价
**回答：如果想把毛利率做到 XX%，原料应该是什么价**

功能：
- 选择 SKU + 规格 + 当前售价 + 目标门店毛利率
- 系统自动计算：目标允许门店成本、当前成本差额
- 区分"可调成本"（原料）与"固定成本"（包材 / 耗材）
- 输出每个可调原料的建议目标单价（理想价 / 可接受价 / 红线价）
- 支持"锁定某些原料不参与调整"

成本拆解规则：
- 固定成本：包材、标配耗材，由 `产品出品表_*` 的包材行识别
- 可调成本：奶浆、果酱、坚果、巧克力、糖浆、半成品，由 `产品配方表_*` 识别

关联数据：
- `产品毛利表_*` — 定价、毛利率
- `产品配方表_*` — 原料组成、用量、单位成本
- `半成品配方表_*` — 半成品原料明细

---

### 模块 3 — 原料价格模拟器
**回答：换个原料价，利润会怎么变**

功能：
- 单原料调价（精确值 / 百分比）
- 按类别批量调价（坚果类 +8%、包材类 -5%）
- 保存为版本（当前版 / 保守采购版 / 理想议价版 / 旺季版）
- 对比任意两个版本：毛利变化、利润变化、受影响 SKU 列表

输出：
- 受影响 SKU 数
- 毛利提升 / 下降最多的前 10 个产品
- 整体原料采购成本变化
- 高风险产品（毛利率降至 X% 以下的产品）

数据来源：
- `总原料成本表`
- `产品配方表_*`
- `产品成本计算表_*`

---

### 模块 4 — 生产计划与备料
**回答：要卖多少、什么时候做、原料够不够**

这是本工具与纯分析工具拉开差距的核心模块。

#### 4a. 生产计划录入
- 按日期录入：产品、规格、数量
- 支持手工逐行添加
- 支持 Excel 批量导入
- 支持复制历史计划（节假日模板）
- 计划类型区分：销量计划 / 生产计划（两者有不同的提前期逻辑）

#### 4b. BOM 自动展开
系统根据配方自动做三级展开：
- **SKU 级** → 主原料需求 + 配料需求
- **主原料级** → 半成品需求（如橙子味风味水）
- **半成品级** → 原料需求（如橙汁、糖浆、基底乳）

#### 4c. 备料规则引擎
| 规则 | 说明 |
|------|------|
| 损耗率 | 实际备料 = 理论需求 × (1 + 损耗率)，按原料类别配置 |
| 安全库存 | 建议备量 = 理论需求 × (1 + 安全库存率) |
| 提前期 | 某原料需提前 N 天采购，系统标注"最晚下单日" |
| 最小采购单位 | 包材按箱、糖浆按瓶，取整不出现 2.3 瓶 |
| 批次生产 | 某些半成品必须按整锅 / 整桶备料 |

#### 4d. 备料计划输出
按日期输出：
- 原料名、单位
- 理论需求、调整后需求、建议备量
- 当前库存（V1 留空，V2 可扩展）
- 缺口量、建议采购量

支持视图：
- 按天 / 按周 / 时间区间汇总
- 按原料分类筛选

#### 4e. 采购建议页
将备料计划翻译为采购语言：
- 下单日期、到货日期
- 原料、数量、单位
- 原因（满足哪天的生产计划）

数据来源：
- `产品出品表_*` — SKU 原料明细
- `产品配方表_*` — 主原料配料
- `半成品配方表_*` — 半成品配料
- `总原料成本表` — 原料单位与价格

---

### 模块 5 — 产品组合评估
**回答：这个组合值不值得做**

功能：
- 左侧选产品 + 填数量
- 右侧实时显示：总销售额、总成本、毛利额、毛利率、净利润
- 实时显示原料种类数 + 产能压力指数
- 切换出厂口径 / 门店口径，全局面联动刷新

支持场景保存：
- 保存方案 A / B / C
- 对比两个方案的利润、原料、产能差异

---

### 模块 6（贯穿）— 数据健康与可信度提示
**回答：这个结果能信多少**

原因：源数据有 103 处计算错误，必须诚实告知用户。

功能：
- 列出所有"计算错误"单元格及影响的 SKU / 原料
- 标记：哪些被置零、哪些是估算、哪些结果需谨慎解读
- 原料缺口预警：某原料无有效单价时，提示"该原料成本为 0，建议核实"

## 4. 核心数据对象

### Product
```
name, category, spec, status
price, cost_factory, cost_store
margin_factory, margin_store
recipe_items: list[RecipeItem]
is_adjustable: bool  # 是否参与定价反推
```

### RecipeItem
```
product_name, ingredient, quantity, unit_cost, total_cost
is_packaging: bool  # 是否为包材/固定成本
```

### Material
```
name, unit, cost, category
is_adjustable: bool
scenarios: dict[str, float]  # 各版本价格
```

### MaterialPriceScenario
```
name, created_at, notes
覆盖表: {原料名: 单价}
```

### MarginTargetRule
```
target_object: Product | Category | 'all'
target_margin: float
lock_ingredients: list[str]
```

### ProductionPlan
```
date, product_name, spec, quantity
plan_type: 'sales' | 'production'
scenario_name: str
```

### PrepRule
```
material_name
loss_rate: float
safety_stock_rate: float
lead_days: int
min_unit: float
batch_size: float | null
```

### MaterialPrepPlan
```
date, material_name, unit
theoretical_qty, adjusted_qty, suggested_qty
purchase_qty, gap
sources: list[str]  # 来源产品列表
```

## 5. 验收标准

| ID | 标准 | 验证方式 |
|----|------|---------|
| AC-1 | Excel 所有 sheet 解析成功，无报错 | `data/loader.py` 运行无异常 |
| AC-2 | SKU 毛利计算与 Excel 原始数据误差 < 0.01 元 | 抽检 5 个 SKU |
| AC-3 | 目标毛利率反推，给出可调原料建议价 | 用木姜子甜橙小杯（目标 72%）验证 |
| AC-4 | 原料调价后，组合毛利实时联动刷新 | 开心果酱价格 +10% 验证毛利变化 |
| AC-5 | 输入 7 天生产计划，< 3 秒输出每日备料表 | pytest 计时 |
| AC-6 | 采购建议输出：日期 + 原料 + 数量 + 单位 | 与人工计算对比 |
| AC-7 | 数据校验报告列出所有计算错误及影响范围 | 与已知 103 处对比 |
| AC-8 | 方案 A / B 对比：利润差、毛利率差、原料需求差 | 边界用例测试 |

## 7. 数据质量问题汇总（来源分析）

| Sheet | 行数 | 计算错误数 | 影响 |
|-------|------|----------|------|
| 总原料成本表 | ~166 | ~82 | 原料成本置零 |
| 产品成本计算表_Gelato | — | ~21 | SKU 成本错误 |
| 合计 | — | **~103** | 需在加载时特殊处理 |

处理策略：
- `data/validator.py` 扫描并记录所有异常
- 计算时该行置零，输出警告，不静默通过
- UI 层显示"此结果包含 N 处估算值"
